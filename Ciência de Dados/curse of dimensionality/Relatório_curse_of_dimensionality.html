1 Introduo

O problema conhecido como curse of dimensionality (maldio da dimensionalidade) ocorre quando o nmero de dimenses de um conjunto de dados aumenta, tornando os pontos cada vez mais esparsos. Isso faz com que a noo de vizinhana perca sentido e prejudica algoritmos que dependem de medidas de distncia, como o K-Nearest Neighbors. Para ilustrar esse efeito, foram realizadas simulaes.

Na Seo 2, um hipercubo de volume unitrio construdo com uma hiperesfera centralizada. medida que a dimenso cresce, o volume da hiperesfera torna-se quase insignificante, mostrando como o espao central se torna vazio em alta dimenso.

Na Seo 3 so criados dois clusters de dados, A e B, separados por uma distncia em um espao p-dimensional e na Seo 4, aplicada uma tcnica de reduo de dimensionalidade, projetando cada observao na linha que conecta os centrides dos clusters.

Por fim, na Seo 6, aplica-se o classificador \(k\)-NN para examinar como a combinao de alta dimensionalidade e diferentes tamanhos de cluster influencia a taxa de erro.

2 Simulando dados em hipercubos e hiperesferas

Podemos estudar o problema de curse of dimensionality considerando como exemplo dados distribudos de maneira uniforme em um hipercubo de volume 1. Considere uma hiperesfera de raio \(r\) centralizada dentro do hipercubo. O volume da hiperesfera de dimenso \(p\) e raio \(r\) dado por:

\[V_p(r) = \frac{\pi^{p/2}}{\Gamma(p/2+1)}r^p \]

De forma que, considerando raio \(r\) (\(r\leq 1/2\)) e dimenso \(p\), a proporo do volume da hiperesfera em relao ao do hipercubo de volume 1 :

\[R_{p, r} = \frac{V_p(r)}{1}\]

calcula_razao <- function(r, p) {
  
  if (r > 1/2) {
    print("Valor de raio invlido, precisa ser <= 1/2")
  } 
  else if (p <= 0 || p != as.integer(p)) {
    print("Dimenso p invlida, precisa ser inteiro positivo")
  } 
  else {
    
    V_pr = (pi^(p/2) * r^p) / gamma(p/2 + 1)
    
    R_pr = V_pr

    return(data.frame(r = r, p = p, Rpr = R_pr))
  }
}

3 Simulando dois clusters

Visando criar um conjunto de dados com dois clusters (A e B) em um espao de \(p\) dimenses, define-se cada cluster em torno de um centride, com as observaes distribudas segundo uma distribuio normal multivariada, permitindo modelar simultaneamente vrias variveis contnuas. Para simplificar a interpretao dos dados, adota-se uma matriz de covarincia identidade, garantindo que cada dimenso seja independente e sem correlao entre si. O cluster B deslocado de forma a garantir que a distncia entre os centrides seja aproximadamente \(s\).

set.seed(12345)

simula_clusters <- function(n, s, p) {
  
  # Validao dos parmentos
  if (n <= 0 || n %% 1 != 0) {
    print("'n' deve ser inteiro positivo")
  }
  else if (s <= 0) {
    print("'s' deve ser positivo")
  }
  else if (p <= 0 || p %% 1 != 0) {
    print("'p' deve ser inteiro positivo")
  }
  else {
  # Cluster A centrado na origem
  mu_A <- rep(0, p)
  # Cluster B deslocado em 's' na primeira dimenso
  mu_B <- c(s, rep(0, p-1))
  
  # matriz Identidade p x p: varincia 1, sem correlao entre variveis
  Sigma <- diag(p)
  
  #Cluster A e B
  A <- mvrnorm(n, mu_A, Sigma)
  B <- mvrnorm(n, mu_B, Sigma)
  
 #Tabela:
  dados <- rbind(A, B) |> 
    as.data.frame()
  colnames(dados) <- paste0("X", 1:p)
  
  dados <- add_column(dados, cluster = rep(c("A","B"), each = n), .before = 1)
  
  return(dados)
  }
}

4 Avaliando os dados simulados

Para avaliar os dados simulados, aplicamos uma funo que reduz os dados de p dimenses para uma nica dimenso, projetando cada observao na direo que conecta os centrides dos clusters A e B. Cada ponto recebe um valor \(proj\) que indica sua posio ao longo dessa linha, permitindo avaliar a separao entre os clusters de forma clara.

O procedimento consiste em:

  1. calcular os centrides de cada cluster;

\[ \mu_A = \frac{1}{n_A} \sum_{i \in A} x_i, \qquad \mu_B = \frac{1}{n_B} \sum_{i \in B} x_i \]

  1. definir a direo entre eles como um vetor unitrio;

O vetor direo entre os centrides :

\[ d = \mu_B - \mu_A \]

O vetor unitrio correspondente :

\[ u = \frac{d}{\|d\|} \]

  1. projetar cada observao subtraindo o centride do cluster A e multiplicando pelo vetor unitrio.

\[ proj(x_i) = (x_i - \mu_A) \cdot u \]

projecao <- function(dados) {
  
  # matriz de preditores
  X <- dados %>% dplyr::select(-cluster)
  
  # CLCULO DOS CENTRIDES
  centroid_A <- dados %>% filter(cluster == "A") %>% dplyr::select(-cluster) %>% colMeans()
  
  centroid_B <- dados %>% filter(cluster == "B") %>% dplyr::select(-cluster) %>% colMeans()
  
  # O vetor de direo
  v <- centroid_B - centroid_A
  
  #vetor unitrio 'u'.
  u <- v / sqrt(sum(v^2))
  
  # PROJEO DOS DADOS
  proj_valores <- as.matrix(X) %>%
    sweep(., 2, centroid_A, "-") %*% u
  
  resultado_proj <- tibble(
    proj = as.vector(proj_valores),
    cluster = dados$cluster
  )
  
  return(resultado_proj)
}

5 Calculando as distncias

5.1 Distncia euclideana entre os centrides

dist_centroides <- function(dados) {
  
  # 1. Calcular os centrides dos clusters "A" e "B"
  centroid_A <- dados %>% filter(cluster == "A") %>% dplyr::select(-cluster) %>% colMeans()
  centroid_B <- dados %>% filter(cluster == "B") %>% dplyr::select(-cluster) %>% colMeans()
  
  # 2. Calcular a distncia euclidiana entre os centrides
  distancia <- sqrt(sum((centroid_A - centroid_B)^2))
  
  # 3. Retornar o valor da distncia
  return(distancia)
}
distancia_centroides <- dist_centroides(dados_simulados)

A funo dist_centroides tem como objetivo calcular a distncia euclidiana entre os centrides dos clusters A e B. Primeiramente, so extradas as observaes de cada grupo e, em seguida, calculadas as mdias de cada varivel, obtendo-se assim os vetores que representam os centrides. A partir desses vetores, aplica-se a frmula da distncia euclidiana em dimenso \(p\):

\[ d(A,B) = \sqrt{ \sum_{j=1}^{p} \left( centroid_A^{(j)} - centroid_B^{(j)} \right)^2 } \]

No caso dos dados simulados, o resultado obtido de aproximadamente \(1,89\) est bem prximo da distncia definida na simulao (\(s = 2\)), e a pequena diferena explicada pelo carter aleatrio da gerao dos dados. Portanto, o clculo confirma que os clusters foram gerados de forma bem separada no espao multidimensional.

5.2 Distncia em pares

dist_pares <- function(dados) {
  
  # 1. Separar os dados em clusters A e B
  dados_A <- dados %>% filter(cluster == "A") %>% dplyr::select(-cluster) 
  dados_B <- dados %>% filter(cluster == "B") %>% dplyr::select(-cluster)

  # 2. Criar uma funo para calcular a distncia euclidiana entre dois pontos
  calc_distancia <- function(ponto1, ponto2) {
    sqrt(sum((ponto1 - ponto2)^2))
  }
  
  # 3. Inicializar uma lista para armazenar os resultados
  resultados <- list()
  
  # 4. Calcular distncias para pares dentro de A
  for(i in 1:(nrow(dados_A) - 1)) {
    for(j in (i + 1):nrow(dados_A)) {
      dist <- calc_distancia(as.numeric(dados_A[i, ]), as.numeric(dados_A[j, ]))
      resultados <- append(resultados, list(data.frame(distancia = dist, tipo = "Dentro de A")))
    }
  }
  
  # 5. Calcular distncias para pares dentro de B
  for(i in 1:(nrow(dados_B) - 1)) {
    for(j in (i + 1):nrow(dados_B)) {
      dist <- calc_distancia(as.numeric(dados_B[i, ]), as.numeric(dados_B[j, ]))
      resultados <- append(resultados, list(data.frame(distancia = dist, tipo = "Dentro de B")))
    }
  }
  
  # 6. Calcular distncias para pares entre A e B
  for(i in 1:nrow(dados_A)) {
    for(j in 1:nrow(dados_B)) {
      dist <- calc_distancia(as.numeric(dados_A[i, ]), as.numeric(dados_B[j, ]))
      resultados <- append(resultados, list(data.frame(distancia = dist, tipo = "Entre")))
    }
  }
  
  # 7. Combinar todos os resultados em um nico data.frame
  resultado_final <- bind_rows(resultados)
  
  return(resultado_final)
}
# Usando a funo com os dados simulados
resultados_distancias <- dist_pares(dados_simulados)

A funo dist_pares tem como objetivo calcular as distncias euclidianas entre todos os pares de pontos do conjunto de dados, separando esses clculos em trs categorias: distncias entre pontos pertencentes apenas ao cluster A, distncias entre pontos pertencentes apenas ao cluster B e distncias entre pontos pertencentes a clusters diferentes (um no A e outro no B). Para isso, a funo primeiro separa os dados em dois subconjuntos, um para o cluster A e outro para o cluster B. Em seguida, calculada a distncia euclidiana entre dois pontos genricos \(x_i\) e \(x_j\), em um espao de dimenso \(p\), dada por:

\[ d(x_i, x_j) = \sqrt{ \sum_{k=1}^{p} \left( x_i^{(k)} - x_j^{(k)} \right)^2 } \]

Depois disso, so criados laos de repetio que percorrem os pares possveis de observaes dentro do cluster A, dentro do cluster B e entre os clusters A e B. Em geral, espera-se que as distncias dentro de A e dentro de B sejam menores (indicando que os pontos de cada cluster esto prximos entre si), enquanto as distncias entre A e B tendem a ser maiores (mostrando que os grupos esto separados no espao).

Para confirmar isso, calculamos a mdia das distncias euclidianas (Tabela 5.1), observamos que os valores dentro de A e dentro de B so relativamente prximos e baixos, indicando que os pontos de cada cluster esto concentrados e prximos entre si. J a mdia das distncias entre A e B consideravelmente maior, o que mostra que os dois clusters esto bem separados.

6 Usando k-NN para classificao

Para avaliar o desempenho do classificador \(k-NN\), foram utilizados os dados simulados criados anteriormente. Os dados foram divididos em conjuntos de treino e teste, e o melhor valor de \(k\) do \(k-NN\), foi determinado por meio de uma validao cruzada de 10 folds no conjunto de treino. No conjunto de teste, a taxa de erro foi calculada como a proporo de observaes classificadas incorretamente.

set.seed(42) 
avalia_knn <- function(n,s,p){
  
  #usando simula_clusters para gerar dados
  dados = simula_clusters(n=n, s=s, p=p)
  dados$cluster = as.factor(dados$cluster)
  
  #separando em treino (70%) e teste (30%)
  set.seed(42)
  indices_treino = sample(1:nrow(dados), size = 0.7*nrow(dados))
  dados_treino = dados[indices_treino,]
  dados_teste = dados[-indices_treino,]
  
  #validao cruzada para encontrar o melhor k (10-fold Cross Validation)
  metodo = trainControl(method = "cv", number = 10)
  grid_k = expand.grid(k = seq(from = 1, to = 19, by=2)) #testando valores impares
  set.seed(42)
  knn_cv = train(cluster~.,data = dados_treino, method = "knn",
                 trControl = metodo, tuneGrid = grid_k)
  
  melhor_k = knn_cv$bestTune$k
  
  treino_x = dados_treino[,-1]
  treino_y = dados_treino[,1]
  teste_x = dados_teste[,-1]
  teste_y = dados_teste[,1]
  
  #previsao no conjunto teste com o melhor k
  previsoes = knn(train = treino_x, test = teste_x, cl = treino_y, k = melhor_k)
  
  taxa_erro = mean(previsoes != teste_y)
  
  resultado = data.frame(erro = taxa_erro, p = p, n = n)
  
  return(resultado)
}

Os experimentos realizados na Seo 2 mostram os efeitos da dimensionalidade nos dados e nos mtodos de anlise. Observou-se que o volume da hiperesfera, na Figura 2.1 e na Tabela 2.1, diminui rapidamente em relao ao hipercubo medida que a dimenso \(p\) aumenta, evidenciando que o espao se torna cada vez mais vazio em altas dimenses um fenmeno da curse of dimensionality.

Figure 7.1: Distribuicao das Distancias em Pares para Diferentes Dimensoes p

Em dimenses baixas, as distncias intra-cluster so menores e bem separadas das inter-cluster, mas medida que \(p\) cresce, todas as distncias aumentam e as curvas se sobrepem cada vez mais. Em dimenses altas, as distribuies ficam praticamente iguais, tornando impossvel distinguir pontos de clusters diferentes.

Em dimenses altas, as curvas se espalham significativamente, e os clusters se misturam, mesmo que os centroides permaneam distintos.

Por fim, o grfico 7.3 mostra que a taxa de erro de classificao aumenta com a dimensionalidade, evidenciando a maldio da dimensionalidade, especialmente para clusters pequenos (\(n=50\)). Em dimenses intermedirias (2050) h um ponto de transio: o erro se estabiliza para clusters grandes, mas continua elevado para clusters pequenos, destacando a importncia do tamanho do cluster em espaos de alta dimenso.

8 Discusso

Os resultados obtidos confirmam os efeitos da maldio da dimensionalidade. medida que a dimenso \(p\) aumenta, as distncias absolutas entre pares de pontos tendem a crescer. Em altas dimenses, a variabilidade relativa entre as distncias diminui, fazendo com que todos os pontos estejam aproximadamente mesma distncia uns dos outros.

A projeo dos dados na direo que conecta os centrides dos clusters mostrou-se uma estratgia til para evidenciar a diferena entre eles. Essa estratgia pode ser justificada pelo fato de que essa direo concentra a maior variao entre os grupos, maximizando a separao das mdias. Assim, ainda que a alta dimensionalidade dificulte a visualizao e a classificao, a projeo preserva a informao mais relevante para distinguir os clusters, facilitando a anlise e reduzindo parcialmente os efeitos da dimensionalidade.

Outro fator relevante a relao entre o nmero de observaes \(n\) por cluster e a dimenso do espao \(p\). Quando \(n\) reduzido, o aumento de \(p\) reala a esparsidade do espao, resultando em maior sobreposio entre as distribuies e, consequentemente, no aumento da taxa de erro do \(k\)-NN. No entanto, para valores mais elevados de \(n\), h mais pontos disponveis prximos ao ponto a ser classificado, o que ajuda a manter o desempenho do \(k\)-NN por mais tempo antes que a maldio da dimensionalidade finalmete prevalea, e contribui para reduzir os erros de classificao em dimenses intermedirias. Contudo, mesmo em cenrios com grandes amostras, o aumento excessivo de \(p\) inevitavelmente compromete a performance do mtodo, dado que a maldio da dimensionalidade no pode ser completamente eliminada.
